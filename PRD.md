# 多链接批量去水印工具 (Watermark-Free Batch) 需求需求与设计报告

## 1. 技术核心原理分析 (Technical Core)

本项目旨在实现类似“轻抖”的功能，通过绕过平台前端限制，直接获取服务端存储的无水印原始资源。

### 1.1 核心链路
1.  **链接提取**：从用户粘贴的混合文案中正则匹配出 `v.douyin.com` 或 `xiaohongshu.com` 的 URL。
2.  **短链还原**：发起 HTTP 请求（禁止重定向），从响应头的 `Location` 获取包含 `ItemID` 的长链接。
3.  **API 请求与签名绕过**：
    *   **抖音**：需构造带 `X-Bogus`、`a-bogus` 等加密签名的请求。
    *   **小红书**：需处理 `Cookie`、`Referer` 及可能的校验参数。
4.  **数据解析**：从返回的 JSON 数据中提取 `play_addr` (视频) 或 `image_list` (图文) 的无水印 CDN 地址。
5.  **前端展示与下载**：将解析结果反馈给用户，提供预览及批量下载功能。

---

## 2. 产品设计方案 (Product Design)

### 2.1 视觉风格
*   **极简主义**：中心化输入框，支持多行输入。
*   **实时反馈**：任务列表展示每个任务的解析进度（待处理 -> 解析中 -> 成功/失败）。
*   **现代交互**：支持深色模式，使用卡片式布局展示解析后的视频封面或图片缩略图。

### 2.2 核心模块设计
*   **智能识别区**：自动过滤干扰文字，仅保留有效链接。
*   **批量任务管理器**：支持并发或依次处理多个解析请求。
*   **媒体预览弹窗**：点击解析成功的任务，可直接在网页内预览无水印视频。
*   **一键导出**：支持“复制所有链接”及“一键下载所有资源”。

## 3. 详细需求清单 (Requirements)

| 模块 | 功能点 | 详细描述 |
| :--- | :--- | :--- |
| **前端输入** | 多链接自动提取 | 用户粘贴包含文字和链接的混合内容，前端正则提取所有有效 URL。 |
| | 任务列表显示 | 列表展示所有提取到的任务，支持手动删除或排序。 |
| **后端解析** | 多平台适配 | 核心逻辑需覆盖：抖音（短视频/图文）、小红书（视频/图文笔记）。 |
| | 异步解析队列 | 采用异步处理机制，防止大量链接导致请求超时。 |
| | 签名服务集成 | 维护或调用最新的签名算法接口，确保解析稳定性。 |
| **结果输出** | 状态动态更新 | 实时显示“解析中”、“成功”、“失败：[原因]”。 |
| | 媒体元数据获取 | 解析成功后显示视频标题、作者、封面图。 |
| | 无水印直链获取 | 提供直接跳转或下载的 CDN 绝对路径。 |

---

## 4. 推荐技术栈 (Tech Stack)

*   **前端 (Frontend)**: React / Vue 3 + Tailwind CSS (快速构建响应式 UI)。
*   **后端 (Backend)**: Node.js (Express) 或 Python (FastAPI) (适合处理并发请求)。
*   **网络请求**: `axios` (前端/后端) 或 `httpx` (Python)。
*   **部署**: 可使用 Docker 容器化部署，方便扩展代理 IP 池。

---

## 5. 开发建议与风险防控
1.  **接口稳定性**：平台 API 变动频繁，建议将解析逻辑模块化，便于快速更新。
2.  **风控应对**：建议集成代理 IP 池（Proxy Pool）以应对平台的频率限制。
3.  **法律合规**：本工具仅限个人学习研究使用，需在页面显著位置添加免责声明。
